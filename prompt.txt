STEP 2 — Refactor chest_site.gd to use the relic pool
======================================================

Replace the current hardcoded Bandage Roll UI logic in _build_ui() so that:

1. On _ready or _build_ui, determine which relics the player does NOT already own:
   var available = []
   for relic_name in RELIC_POOL.keys():
       if not GameState.relics.has(relic_name):
           available.append(relic_name)

2. If available is empty (player owns all relics):
   - Show "Chest Empty" or "No new relics"
   - Disable the Take button
   - Set a var like _chosen_relic = "" so _on_take_pressed does nothing

3. If available is NOT empty:
   - Pick one at random: var _chosen_relic = available[randi() % available.size()]
   - Display that relic's name and description from RELIC_POOL[_chosen_relic]
   - Enable the Take button

4. In _on_take_pressed():
   - Instead of hardcoding "Bandage Roll", use _chosen_relic:
     GameState.relics.append(_chosen_relic)
     print("Relic acquired: ", _chosen_relic)
   - Disable the button after taking

Keep the Leave button exactly as it is now.

Store _chosen_relic as a script-level var (like the existing _take_btn var).

Where: res://Scripts/gamer_manager_node.gd, in the start_turn() function

Currently there is this block (around line 349):

Add Iron Shield right after that block, same pattern:
