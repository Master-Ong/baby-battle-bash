## CURRENT STATE — WHAT WORKS ✅

**Core Loop:**
- Full roguelike loop: Main Menu → Animal Select → Roadmap → Combat → Reward → Roadmap
- Lose screen with New Run and Main Menu
- Run Complete screen (after boss) with clean Main Menu return
- GameState.reset_run() works cleanly between runs

**Map 1 Content:**
- 7 roadmap nodes: Combat, Rest, Combat, Combat, Shop, Chest, Boss
- Rest Site heals 15 HP
- Chest Site gives Bandage Roll relic
- Bandage Roll gives 5 Block on turn 1
- Boss node: Baby Dragon (90 HP, 16 ATK)

**Gold System:**
- GameState.gold starts at 0
- Normal combat reward awards +10 gold
- Gold accumulates during run
- Gold resets on new run
- Boss victory does not award gold

**Shop System:**
- Shop opens from roadmap
- Market tab: 3 unique card offers (no visible duplicates)
- Adopt costs 10g (testing price)
- Reroll costs 1g per slot (testing price)
- Purchased cards enter GameState.reward_cards
- Return to Roadmap preserves run state

**Fusion/Evolution v0:**
- Training Camp button in Shop (bottom-left)
- Opens overlay panel showing available fusions
- Rules: 2 Bunny → Bunny+, 2 Turtle → Turtle+, 2 Dog → Dog+
- Uses exact card_name matching (Bunny+ does NOT count as Bunny)
- Removes 2 base cards, adds 1 upgraded card from CardDatabase
- Panel refreshes after fusion
- Free (no gold cost)
- Only works on reward/shop/test-selector animals (NOT starter deck copies)
- **STATUS: Code verified, needs gameplay testing**

**Shared CardDatabase:**
- res://Scripts/CardDatabase.gd
- Static functions: get_animals(), get_colors(), get_adjectives(), get_attacks(), get_skills(), get_upgraded_animals()
- Rewards and Shop both use shared definitions
- Total pool: 3 animals, 3 colors, 6 adjectives, 5 attacks, 5 skills, 3 upgraded animals

**Animal Select DevDock (Test Tools):**
- Container-based layout at bottom (no clipping)
- 3 rows: Animals, Adjectives, Verbs
- 19 toggle buttons total
- Tooltips show card descriptions on hover
- Selected cards append to starter deck
- Scene-tree-first design (editable in Godot editor)

**Combat Systems:**
- Combat end-state handling is stable (combat_ended guard, deferred outcome check)
- Block system works (player_defense consumed before HP)
- Field animals with 0 HP are removed to graveyard
- Negative HP clamped to 0 for display
- field_slots is the source of truth
- Card HP mutation bug FIXED (cards cloned when building combat deck)

**Card Pool:**
- Animals: Bunny (4/12), Turtle (2/20), Dog (8/10)
- Upgraded: Bunny+ (6/16), Turtle+ (4/28), Dog+ (12/14)
- Colors: Blue (draw 1), Red (+2 ATK buff), Green (heal 3)
- Adjectives: BIG, FAST, TOUGH, SHARP, SMART, MAGIC
- Attacks: Strike, Scratch, Bite, Dash (4 dmg + draw 1), Pounce, Roar
- Skills: Defend, Hide, Run Away, Block, Curl Up, Guard, Scout (1 block + draw 1), Sneak (4 block + draw 1)

**Mob Pool:**

## WORKFLOW RULES

**ChatGPT → Claude.ai → Claude Code:**
1. ChatGPT drafts feature ideas
2. Claude.ai verifies against real project files
3. Claude Code builds only after verification
4. Never send raw ChatGPT prompts directly to CC

**Patch discipline:**
- Smallest safe patch only
- One system at a time
- Long prompts go through prompt.txt
- Run CC with: `claude -p prompt.txt`

**Testing before building:**
- Always verify existing features work before expanding
- Test Fusion v0 before building next feature

**Screenshot validation (required when MCP disconnects):**
The Godot MCP connection is unstable and drops frequently. When it disconnects:
1. Send prompt to Claude Code
2. CC builds the changes
3. **Nicky takes a screenshot of the result in Godot**
4. **Nicky pastes the screenshot in Claude.ai chat**
5. Claude verifies the result visually
6. Then proceed to next task

This lets us confirm changes worked ("DevDock buttons appeared", "Training Camp panel opened") without needing live file access.
